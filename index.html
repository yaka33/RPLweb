A Lightweight Trust Management Framework in
the Networking Layer of IoT Devices
25_26J_009

Exploring the frontiers of technology to solve real-world problems with cutting-edge solutions.

Project Abstract

A fully distributed, lightweight security framework for RPL-based IoT sensor networks that detects, classifies, and responds to routing attacks — without a centralized server, heavyweight cryptography, or significant computational overhead.

Modern IoT networks built on the RPL routing protocol are critically vulnerable to sophisticated routing attacks such as Sinkhole and Selective Forwarding. Existing solutions either demand excessive computational resources or rely on centralized architectures that introduce single points of failure — making them impractical for constrained IoT devices operating on limited RAM, CPU, and battery.

This research presents a four-component, fully distributed trust management framework running entirely on each individual sensor node using Contiki-NG. Every node independently computes multi-dimensional trust scores, monitors network behaviour through statistical intrusion detection, participates in distributed consensus-based isolation, and enforces trust-aware routing decisions through a custom RPL Objective Function. The result is a cohesive, self-healing security layer that requires no dedicated security server and degrades gracefully under energy constraints.

Overview

The Vision

To create an IoT network that polices itself — where every node is simultaneously a sensor, a guardian, and a decision-maker, collectively maintaining network integrity without any centralised authority.

The Mission

To implement a production-grade, resource-aware security framework for IEEE 802.15.4 IoT networks that detects Sinkhole and Selective Forwarding attacks in real time, isolates malicious nodes through distributed consensus, and enforces trust-aware routing — all within the strict constraints of embedded IoT hardware running Contiki-NG.

Key Impact

  • Zero single point of failure — every node runs all four security components independently
  • Dual-threat detection — simultaneously identifies Sinkhole and Selective Forwarding attacks within a 10-second observation window
  • Energy-aware operation — trust computation frequency adapts dynamically to battery level across four energy modes, extending node lifetime by over 25%
  • Self-healing network — isolated nodes are automatically reinstated after sustained good behaviour, preventing permanent network degradation
  • No core modifications — integrates into the RPL stack entirely through compile-time hooks, requiring no changes to Contiki-NG source files

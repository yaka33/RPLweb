Research Domain

This research focuses on improving trust and security in IoT networks, particularly in environments where data is transmitted through untrusted or intermediate nodes. IoT devices are often resource-constrained and operate in dynamic environments, making them vulnerable to attacks such as data manipulation, selective forwarding, and malicious node behavior.

The study explores how lightweight trust mechanisms can be integrated into the networking layer to ensure reliable communication while maintaining energy efficiency. The proposed solution aims to enhance routing decisions by incorporating trust metrics, enabling IoT systems to operate securely without adding significant computational overhead.

Literature Survey

Existing research highlights several trust-based routing mechanisms for IoT and RPL networks. Studies such as THC-RPL (2022) introduced trust computation techniques to detect malicious nodes, while FSRW (2024) applied fuzzy logic and optimization algorithms to improve routing reliability. Additionally, recent approaches like RSOF (2025) focus on combining trust with routing metrics such as ETX to enhance decision-making.

However, many existing solutions rely on complex computations, high energy consumption, or static trust models, making them unsuitable for constrained IoT devices. While these methods improve security, they often compromise network lifetime and scalability.

Research Gap

Despite advancements in trust-based routing, several limitations remain:

  • Lack of energy-efficient trust computation mechanisms
  • Heavy reliance on complex algorithms unsuitable for low-power devices
  • Absence of adaptive trust update mechanisms based on network conditions
  • Limited integration of multiple trust metrics in a lightweight manner

These gaps highlight the need for a lightweight, adaptive, and energy-aware trust management system that can operate efficiently in real-world IoT environments.

Research Problem

IoT networks often rely on intermediate nodes to forward data, but these nodes may behave maliciously or unreliably. Existing routing protocols such as RPL do not adequately evaluate the trustworthiness of nodes, leading to potential data loss, manipulation, or security breaches.

"The key problem addressed in this research is: How to design a lightweight and energy-efficient trust management framework that ensures reliable data transmission in IoT networks while minimizing computational overhead?"

Research Objectives

Main Objective:

To develop a lightweight trust management framework for IoT networks that enhances secure and reliable data transmission.

Specific Objectives:

  • Design lightweight trust metrics (Residual Energy, Temperature, Historical Observation, Trust Accuracy)
  • Develop an adaptive trust computation mechanism
  • Implement energy-aware trust update scheduling
  • Integrate trust into routing decisions (RPL-based)
  • Evaluate performance using simulation tools (Cooja / Contiki-NG)

Methodology

The proposed system introduces a Trust Computation Engine (TCE) that evaluates node behavior using multiple lightweight metrics. Each node calculates a trust score based on factors such as residual energy, historical performance, and environmental conditions.

  • Metric Collection: Nodes monitor parameters such as energy usage, packet forwarding behavior, and temperature.
  • Trust Calculation: A weighted formula is used to compute trust values dynamically.
  • Adaptive Scheduling: Trust computation frequency is adjusted based on node energy levels to reduce overhead.
  • Routing Integration: Trust values are incorporated into RPL routing decisions to avoid unreliable nodes.
  • Simulation & Evaluation: The system is tested in Contiki-NG/Cooja to measure performance metrics such as Packet Delivery Ratio (PDR), latency, and energy consumption.

Technologies Used

  • Contiki-NG: IoT operating system for simulation
  • Cooja Simulator: Network simulation environment
  • C Programming: Implementation of trust algorithms
  • RPL Protocol: Routing protocol for IoT networks
  • Energest Module: Energy consumption tracking
  • Simple UDP: Communication between nodes
